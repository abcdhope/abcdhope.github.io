golang单元测试教程

参考：Go Test 单元测试简明教程 | 快速入门 | 极客兔兔 (geektutu.com)

单元测试的作用为：测试单个函数/方法；做基准测试；调用数据库访问等的方法等。

简单例子

Go语言推荐测试文件和原代码文件放在一块（两个文件可以分离开，需要import导入源文件所在的包，使用接口时，也需要使用“package_name.API”的方式引用。），且测试文件以_test.go结尾，例如：

exer1/
	| -- main.go
	| -- calc_test.go

两个文件的代码如下：

//main.go
func Add(a int, b int) int {
	return a + b
}

func Mul(a int, b int) int {
	return a * b
}
//calc_test.go
func TestCalc(t *testing.T) {
	if ans := Add(1, 2); ans != 3 {
		t.Errorf("1 + 2 expected be 3, but %d got", ans)
	}
	if ans := Add(-10, -20); ans != -30 {
		t.Errorf("-10 + -20 expected be -30, but %d got", ans)
	}
}

这两个文件同属于一个包下，测试函数的名称都是以TestXxx的格式来命名。

测试用的参数有且只有一个，对于TestCalc函数参数为t *testing.T，而基准测试的参数为*testing.B，TestMain的参数是*testing.M类型。

运行go test，会运行该包下的所有的测试用例，包括不同文件名的测试文件：

> go test
PASS
ok      exer1   0.197s

运行go test -v可以查看不同测试用例的运行情况：

> go test -v
=== RUN   TestCalc
--- PASS: TestCalc (0.00s)
=== RUN   TestMul
--- PASS: TestMul (0.00s)
PASS
ok      exer1   0.256s

添加-run参数可以运行指定测试用例的名字：

> go test -run TestMul -v
=== RUN   TestMul
--- PASS: TestMul (0.00s)
PASS
ok      exer1   0.040s

子测试

子测试的作用在于能在一个测试函数中创建不同测试用例，使用t.Run来创建子测试用例，t.Run第一个参数为测试用例的名字，第二个参数为测试用例的匿名函数：

func TestMul1(t *testing.T) {
	t.Run("pos", func(t *testing.T) {
		if Mul(2, 3) != 6 {
			t.Fatal("fail")
		}
	})

	t.Run("neg", func(t *testing.T) {
		if Mul(2, -3) != -6 {
			t.Fatal("fail")
		}
	})
}

在该函数中，使用了t.Fatal，该函数与之前t.Errorf的不同在于前者遇错会停止，而后者遇错后会继续运行其他测试用例。

输入go test -run TestMul/pos -v可运行函数中指定名字的测试用例：

> go test -run TestMul1/pos -v
=== RUN   TestMul1
=== RUN   TestMul1/pos
--- PASS: TestMul1 (0.00s)
    --- PASS: TestMul1/pos (0.00s)
PASS
ok      exer1   0.210s

如果不指定名字则会运行该函数中的所有测试用例：

> go test -run TestMul1 -v    
=== RUN   TestMul1
=== RUN   TestMul1/pos
=== RUN   TestMul1/neg
--- PASS: TestMul1 (0.00s)
    --- PASS: TestMul1/pos (0.00s)
    --- PASS: TestMul1/neg (0.00s)
PASS
ok      exer1   0.040s

遇到多个子测试的场景，可以采用for循环来定义测试用例：

func TestMul(t *testing.T) {
	cases := []struct {
		Name           string
		A, B, Expected int
	}{
		{"pos", 2, 3, 6},
		{"neg", 2, -3, -6},
		{"zero", 2, 0, 0},
	}

	for _, c := range cases {
		t.Run(c.Name, func(t *testing.T) {
			if ans := Mul(c.A, c.B); ans != c.Expected {
				t.Fatalf("%d * %d expected %d, but %d got",
					c.A, c.B, c.Expected, ans)
			}
		})
	}
}

帮助函数

有些时候不同的测试用例的功能相同但传入的数值不同，此时可以将功能抽离出来用于创建一个函数：

type calcCase struct{ A, B, Expected int }

func createMulTestCase(t *testing.T, c *calcCase) {
	// t.Helper()
	if ans := Mul(c.A, c.B); ans != c.Expected {
		t.Fatalf("%d * %d expected %d, but %d got",
			c.A, c.B, c.Expected, ans)
	}

}

func TestMul2(t *testing.T) {
	createMulTestCase(t, &calcCase{2, 3, 6})
	createMulTestCase(t, &calcCase{2, -3, -6})
	createMulTestCase(t, &calcCase{2, 0, 1}) // wrong case
}

这里故意写错一个测试用例，运行用例，此时会报告出现错误的位置：

> go test -run TestMul2 -v
=== RUN   TestMul2
    calc_test.go:39: 2 * 0 expected 1, but 0 got
--- FAIL: TestMul2 (0.00s)
FAIL
exit status 1
FAIL    exer1   0.210s

但是这里有三个测试用例，而报告的位置是函数里面的错误，因此无法得知确切的位置。

此时可在帮助函数内部引入t.Helper()，该方法可以输出哪个测试用例出现错误：

func createMulTestCase(t *testing.T, c *calcCase) {
	t.Helper()
	if ans := Mul(c.A, c.B); ans != c.Expected {
		t.Fatalf("%d * %d expected %d, but %d got",
			c.A, c.B, c.Expected, ans)
	}

}
输出为：
> go test -run TestMul2 -v
=== RUN   TestMul2
    calc_test.go:48: 2 * 0 expected 1, but 0 got
--- FAIL: TestMul2 (0.00s)
FAIL
exit status 1
FAIL    exer1   0.225s

关于 helper 函数的 2 个建议：

  • 不要返回错误， 帮助函数内部直接使用 t.Error 或 t.Fatal 即可，在用例主逻辑中不会因为太多的错误处理代码，影响可读性。
  • 调用 t.Helper() 让报错信息更准确，有助于定位。

setup 和 teardown

setp和teardown函数可用于不同测试用例的前后逻辑相同的情况下，setup负责前逻辑，teardown负责后逻辑，然后统一由TestMain来进行调度:

func setup() {
	fmt.Println("Before all tests")
}

func teardown() {
	fmt.Println("After all tests")
}

func Test1(t *testing.T) {
	fmt.Println("I'm test1")
}

func Test2(t *testing.T) {
	fmt.Println("I'm test2")
}

func TestMain(m *testing.M) {
	setup()
	code := m.Run()
	teardown()
	os.Exit(code)
}

运行结果：

> go test -v
Before all tests
=== RUN   Test1
I'm test1
--- PASS: Test1 (0.00s)
=== RUN   Test2
I'm test2
--- PASS: Test2 (0.00s)
PASS
After all tests
ok      _/D_/go_exer/mygo/testExer/exer2        0.227s

调用 m.Run() 触发所有测试用例的执行，并使用 os.Exit() 处理返回的状态码，如果不为0，说明有用例失败。

如果出现错误，则其他测试用例不受影响：

func Test1(t *testing.T) {
	var t1 int = 5
	var t2 int = 4
	if tt := t1 + t2; tt != 8 {
		t.Fatal("error")
	}
	fmt.Println("I'm test1")
}

func Test2(t *testing.T) {
	fmt.Println("I'm test2")
}

输出为：
> go test -v
Before all tests
=== RUN   Test1
    setup_test.go:21: error
--- FAIL: Test1 (0.00s)
=== RUN   Test2
I'm test2
--- PASS: Test2 (0.00s)
FAIL
After all tests
exit status 1
FAIL    _/D_/go_exer/mygo/testExer/exer2        0.039s

可以看到Test2正常运行。

网络测试

TCP/HTTP

如果需要测试API的处理函数handler是否有效，可以创建网络连接来测试接口函数：

func helloHandler(w http.ResponseWriter, r *http.Request) {
	w.Write([]byte("hello world"))
}

创建网络连接，添加路由，访问路径来获取结果：

//错误处理
func handleError(t *testing.T, err error) {
	t.Helper()
	if err != nil {
		t.Fatal("failed", err)
	}
}

func TestConn(t *testing.T) {
	ln, err := net.Listen("tcp", ":0")
	handleError(t, err)
	http.HandleFunc("/hello", helloHandler)
	go http.Serve(ln, nil)

	resp, err := http.Get("http://" + ln.Addr().String() + "/hello")
	handleError(t, err)

	defer resp.Body.Close()
	body, err := ioutil.ReadAll(resp.Body)
	handleError(t, err)

	if string(body) != "hello world" {
		t.Fatal("expected hello world, but got", string(body))
	}

}

测试结果：

> go test -run TestConn -v
Before all tests
=== RUN   TestConn
--- PASS: TestConn (0.00s)
PASS
After all tests
ok      _/D_/go_exer/mygo/testExer/exer2        0.059s

httptest

利用httptest来模拟请求对象和响应对象，更快地测试API：

func TestHttptest(t *testing.T) {
	req := httptest.NewRequest("GET", "http://example.com/foo", nil)
	w := httptest.NewRecorder()
	helloHandler(w, req)
	bytes, _ := ioutil.ReadAll(w.Result().Body)
	if string(bytes) != "hello world" {
		t.Fatal("expected hello world, but got", string(bytes))
	}
}

Benchmark基准测试

Benchmark的定义如下：

func BenchmarkXxx(b *testing.B){
    // ...
}

名字以Benchmark开头，后面为自定义的名字，输入参数为*testing.B，执行基准测试时，需要添加-bench参数，该参数利用正则表达式来指定基准测试函数名字，.表示匹配所有基准测试函数，-benchmem命令行标志参数将在报告中包含内存的分配数据统计。

func BenchmarkHello(b *testing.B) {
	for i := 0; i < b.N; i++ {
		fmt.Sprintln("hello")
	}
}

测试：

> go test -v -run="none" -bench . -benchmem
goos: windows
goarch: amd64
cpu: AMD Ryzen 7 4800H with Radeon Graphics
BenchmarkHello
BenchmarkHello-16       18321868                64.48 ns/op            8 B/op          1 allocs/op
PASS
ok      _/D_/go_exer/mygo/testExer/exer2        1.318s

上述命令可以不运行单元测试，只运行所有的基准测试，等价于go test -v -run=^$ -bench . -benchmem。

基准测试报告每一列值对应的含义如下：

type BenchmarkResult struct {
    N         int           // 迭代次数
    T         time.Duration // 基准测试花费的时间
    Bytes     int64         // 一次迭代处理的字节数
    MemAllocs uint64        // 总的分配内存的次数
    MemBytes  uint64        // 总的分配内存的字节数
}

因此，上述参数的意思为：

BenchmarkHello-16       18321868                64.48 ns/op            8 B/op          1 allocs/op

1、BenchmarkHello-16 表示当前系统有16个逻辑CPU
2、18321868 表示运行了18321868次
3、nx/op 表示每次运行耗时，纳秒
4、B/op 表示每次运行使用了内存的字节数
5、allocs/op 表示每次运行分配内存的次数
