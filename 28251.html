7天用Go从零实现Web框架 - Gee

Gee 是一个模仿 gin 实现的 Web 框架，Go Gin简明教程可以快速入门。

第一天 前置知识

目标：

  • 简单介绍net/http库以及http.Handler接口。
  • 搭建Gee框架的雏形，代码约50行。

标准库启动web

利用net/http库来启动web，以一个简单例子来进行描述：

func main() {
	//设置路由
	/*
		调用 http.HandleFunc 实现了路由和Handler的映射，也就是只能针对具体的路由写处理逻辑。
	*/
	http.HandleFunc("/", indexHandler)
	http.HandleFunc("/hello", helloHandler)
	//监听端口
	/*
		第一个参数是地址，:8080表示在 8080 端口监听。
		而第二个参数则代表处理所有的HTTP请求的实例，nil 代表使用标准库中的实例处理。
	*/
	log.Fatal(http.ListenAndServe(":8080", nil))
}

/*
http.ResponseWriter:写入接口
*http.Request:http请求
该函数的作用为：将req的地址写入w
*/
func indexHandler(w http.ResponseWriter, req *http.Request) {
	//%q 双引号围绕的字符串，由Go语法安全地转义 Printf("%q", "Go语言") "Go语言"
	fmt.Fprintf(w, "url:%q\n", req.URL.Path)
}

/*
该函数的作用是：打印请求头
*/
func helloHandler(w http.ResponseWriter, req *http.Request) {
	//Header为键值对
	for k, v := range req.Header {
		fmt.Fprintf(w, "Header[%q]:%q\n", k, v)
	}
}

上述代码的作用为：利用http.HandleFunc设置两个路由/和/hello，用http.ListenAndServe来监听端口8080并处理这两个路由，处理方法分别为indexHandler和helloHandler。

输出为：

C:\Users\11914>curl http://localhost:8080/
url:"/"

C:\Users\11914>curl http://localhost:8080/hello
Header["User-Agent"]:["curl/7.79.1"]
Header["Accept"]:["*/*"]

实现http.Handler接口

http.ListenAndServe的第二个参数类型为Handler接口类型：

package http

type Handler interface {
    ServeHTTP(w ResponseWriter, r *Request)
}

func ListenAndServe(address string, h Handler) error

若第二个参数传入的值为nil，则会默认使用DefaultServeMux（未深入了解）。

1.http.HandleFunc将pattern及我们自定义的handler存储在DefaultServeMux的一个map中。
2.当http.ListenAndServe的handler为nil时，系统会从DefaultServeMux存储信息的map中匹配pattern获取对应的handler，进而处连接请求。

我们可以重定义Handler接口的函数ServeHTTP来自己处理请求，而不需要调用HandleFunc。代码如下所示：

type Engine struct{}

func main() {
	e := new(Engine)
	//第二个参数为Handler接口类型，其内的函数为ServeHTTP函数。
	log.Fatal(http.ListenAndServe(":8080", e))

}

/*
自由定义路由映射的规则
*/
func (*Engine) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	switch r.URL.Path {
	case "/":
		fmt.Fprintf(w, "url:%q\n", r.URL.Path)
	case "/hello":
		//Header为键值对
		for k, v := range r.Header {
			fmt.Fprintf(w, "Header[%q]:%q\n", k, v)
		}
	default:
		fmt.Fprintf(w, "404 NOT FOUND: %s\n", r.URL)
	}
}

上述代码的作用是：定义空结构体Engine，该结构体定义ServeHTTP方法以处理http请求。ServeHTTP函数针对不同的请求地址利用switch语句来作出相应的处理。

Gee框架的雏形

最终的代码结构目录为：

gee/
  |--gee.go
  |--go.mod
main.go
go.mod

其中go.mod使能包

gee.go的代码如下：

package gee

import (
	"fmt"
	"net/http"
)

//定义路由处理方法
type HandlerFunc func(http.ResponseWriter, *http.Request)

//定义路由集合以及对应的处理方法
type Engine struct {
	router map[string]HandlerFunc
}

// 初始化结构体
func New() *Engine {
	return &Engine{router: make(map[string]HandlerFunc)}
}

//将对应的路由添加至结构体中
func (engine *Engine) addRoute(method string, pattern string, handler HandlerFunc) {
	key := method + "-" + pattern
	engine.router[key] = handler
}

//请求方法
func (engine *Engine) GET(pattern string, handler HandlerFunc) {
	engine.addRoute("GET", pattern, handler)
}

func (engine *Engine) POST(pattern string, handler HandlerFunc) {
	engine.addRoute("POST", pattern, handler)
}

//定义处理接口
func (engine *Engine) ServeHTTP(w http.ResponseWriter, req *http.Request) {
	key := req.Method + "-" + req.URL.Path
	if handler, ok := engine.router[key]; ok {
		handler(w, req)
	} else {
		fmt.Fprintf(w, "404 NOT FOUND: %s\n", req.URL)
	}
}

// 监听端口
func (engine *Engine) Run(addr string) (err error) {
	return http.ListenAndServe(addr, engine)
}

该代码的作用为：定义处理方法类型HandlerFunc，该类型可囊括所有处理方法；定义结构体Engine，该结构体只有一个属性，该属性用于存储不同请求类型（GET、POST等）的请求地址的处理方法，并且该结构体定义一系列方法，例如addRoute方法来扩充router集合，GET和POST方法用于调用addRoute方法，ServeHTTP用于处理请求，Run方法用于监听端口。

主函数为：

func main() {
	r := gee.New()
	r.GET("/", func(w http.ResponseWriter, req *http.Request) {
		fmt.Fprintf(w, "URL.Path = %q\n", req.URL.Path)
	})

	r.GET("/hello", func(w http.ResponseWriter, req *http.Request) {
		for k, v := range req.Header {
			fmt.Fprintf(w, "Header[%q] = %q\n", k, v)
		}
	})

	r.Run(":9999")
}

New函数表示创建一个结构体，然后该结构体来调用一系列方法，例如GET方法来创建处理函数和请求，Run启动端口。

第二天 上下文

目标：

  • 将路由(router)独立出来，方便之后增强。
  • 设计上下文(Context)，封装 Request 和 Response ，提供对 JSON、HTML 等返回类型的支持。

Context的作用在于对一系列操作进行封装，例如提交表单、查询键值对、设置http头等。Context结构体包含了请求/响应模型以及对应的方法，例如请求模型中的提交表单、查询键值对，响应模型中的设置http头、设置状态码、编码JSON数据。

context.go:
//存放键值对
type H map[string]interface{}

type Context struct {
	Writer http.ResponseWriter
	Req    *http.Request
	// 请求信息
	Path   string //请求路径
	Method string //请求类型，GET、POST等
	//返回信息
	StatusCode int // 回应代码
}

// 初始化结构体
func NewContext(w http.ResponseWriter, req *http.Request) *Context {
	return &Context{
		Writer: w,
		Req:    req,
		Path:   req.URL.Path,
		Method: req.Method,
	}
}

//获取网页?后面的信息
func (c *Context) PostForm(key string) string {
	return c.Req.FormValue(key)
}

//获取地址?后面的键值对
func (c *Context) Query(key string) string {
	return c.Req.URL.Query().Get(key)
}

//状态码
func (c *Context) Status(code int) {
	c.StatusCode = code
	//WriteHeader sends an HTTP response header with the provided status code.
	c.Writer.WriteHeader(code)
}

//设置http头
func (c *Context) SetHeader(key string, value string) {
	c.Writer.Header().Set(key, value)
}

//设置头部、状态码、主体
func (c *Context) String(code int, format string, values ...interface{}) {
	c.SetHeader("Content-Type", "text/plain")
	c.Status(code)
	c.Writer.Write([]byte(fmt.Sprintf(format, values...)))
}

//JSON编码
func (c *Context) JSON(code int, obj interface{}) {
	c.SetHeader("Content-Type", "application/json")
	c.Status(code)
	encoder := json.NewEncoder(c.Writer)
	if err := encoder.Encode(obj); err != nil {
        panic(err)
	}
}

//编写主体中的数据
func (c *Context) Data(code int, data []byte) {
	c.Status(code)
	c.Writer.Write(data)
}

//编写主体
func (c *Context) HTML(code int, html string) {
	c.SetHeader("Content-Type", "text/html")
	c.Status(code)
	c.Writer.Write([]byte(html))
}

router.go:
//定义路由处理方法
type HandlerFunc func(http.ResponseWriter, *http.Request)

//定义路由集合以及对应的处理方法
type router struct {
	handlers map[string]HandlerFunc
}

//初始化
func NewRouter() *router {
	return &router{handlers: make(map[string]HandlerFunc)}
}

//添加路由
/*
method:请求方法
pattern:路径
handler:路由处理方法
*/
func (r *router) AddRouter(method string, pattern string, handler HandlerFunc) {
	log.Printf("Route %4s - %s", method, pattern)
	key := method + "-" + pattern
	r.handlers[key] = handler
}

//上下文信息
func (r *router) handle(c *Context) {
	key := c.Method + "-" + c.Path
	if handler, ok := r.handlers[key]; ok {
		handler(c.Writer, c.Req)
	} else {
		//404
		c.String(http.StatusNotFound, "404 NOT FOUND: %s\n", c.Path)
	}
}

然后对路由方法进行封装，并添加提交方法（GET、POST）。

//定义路由处理方法
type HandlerFunc func(*Context)

//定义路由集合以及对应的处理方法
type Engine struct {
	router *router
}

//初始化Engine
func New() *Engine {
	return &Engine{router: NewRouter()}
}

func (engine *Engine) addRoute(method string, pattern string, handler HandlerFunc) {
	engine.router.AddRouter(method, pattern, handler)
}

func (engine *Engine) GET(pattern string, handler HandlerFunc) {
	engine.addRoute("GET", pattern, handler)
}

func (engine *Engine) POST(pattern string, handler HandlerFunc) {
	engine.addRoute("POST", pattern, handler)
}

func (engine *Engine) Run(addr string) (err error) {
	return http.ListenAndServe(addr, engine)
}

func (engine *Engine) ServeHTTP(w http.ResponseWriter, req *http.Request) {
	c := NewContext(w, req)
	engine.router.handle(c)
}

Engine结构体对router结构体进一步封装，并添加GET、POST方法，且添加ServeHTTP方法来启动web。

第三天 前缀树路由

前缀树的意思就是按照顺序来存储值，例如有以下路由：

  • /:lang/doc
  • /:lang/tutorial
  • /:lang/intro
  • /about
  • /p/blog
  • /p/related

用前缀树来定义这些路由：

从图中可以看到树分为三层：根节点/，一层节点/:lang、/about、/p，二层节点/doc、/tutorial、/intro、/blog、/related。

现在来实现前缀树，前缀树拥有的功能有：插入节点、查询节点。

  • 定义树结构

    定义树结构需要定义当前节点和子节点，子节点的类型为树类型（为了再延续子节点），以及完整路由（即完整的路径）和标志位（判断该节点是否含有 : 或 *）。

    //树结构
type node struct {
	pattern  string  //待匹配路由，例如 /p/:lang
	part     string  //路由中的一部分，例如 :lang
	children []*node //子节点，例如 [doc, tutorial, intro]
	isWild   bool    //是否精确匹配，part 含有 : 或 * 时为true
}

  • 插入节点

    插入节点的思路为：首先遍历第一层节点，若没存在对应的节点则将该节点插入至这一层，然后根据这一个节点来递归插入子节点。

    //插入节点
/*
pattern:完整路由，用于查询是否已经遍历完毕，例如/p/:lang/doc
parts:路由集合，例如/p/:lang/doc的集合为[p :lang doc]
height:初始为0，逐步增加至完整路由的长度，例如[p :lang doc]的长度为3
*/
func (n *node) insert(pattern string, parts []string, height int) {
	//当输入长度与完整路由的长度相同则表明插入完成
	if len(parts) == height {
		n.pattern = pattern
		return
	}

	//当前路由节点
	part := parts[height]

	//查找树中是否存在该节点
	child := n.matchChild(part)

	//如果树中不存在该节点则创建一个新节点并插入
	if child == nil {
		child = &node{part: part, isWild: part[0] == '*' || part[0] == ':'}
		n.children = append(n.children, child) //将子节点添加进树中
	}

	//继续插入下一个节点
	child.insert(pattern, parts, height+1)

}

    其中，matchChild函数的作用是查找符合条件的子节点，若有则表示该节点在树中存在并返回该节点，没有则返回空值nil。

    //匹配出子节点，用于将节点插入到树中
func (n *node) matchChild(part string) *node {
	//判断子节点中是否存在对应的节点，若有则返回那个子节点
	for _, child := range n.children {
		if part == child.part || child.isWild {
			return child
		}
	}
	return nil
}

  • 查询节点

    查询节点的思路为：由于查询的节点在树中存在多个，因此需要查找所有符合条件的节点并通过数组进行存储，之后在一个一个进行递归查询下一个子节点。

    //查找节点

func (n *node) search(parts []string, height int) *node {
	//判断是否遍历至目标节点
	if len(parts) == height || strings.HasPrefix(n.part, "*") {
		//判断是否遍历到"*"或":"且数组中的节点没有遍历完
		if n.pattern == "" {
			return nil
		}
		return n
	}
	part := parts[height]
	//查找所有符合条件的子节点
	children := n.matchChildren(part)
	for _, child := range children {
		//查询子节点是否存在子节点
		result := child.search(parts, height+1)
		if result != nil {
			return result
		}
	}
	return nil
}

    其中matchChildren函数为存储所有符合条件的节点。

    //匹配出子节点，用于将节点插入到树中
func (n *node) matchChild(part string) *node {
	//判断子节点中是否存在对应的节点，若有则返回那个子节点
	for _, child := range n.children {
		if part == child.part || child.isWild {
			return child
		}
	}
	return nil
}

定义了树结构和方法后，再更新路由方法：

首先是路由结构增添以请求方法为根节点的树结构；

随后就是增加解析路由的方法，即将完整路由分割成路由集合，例如/p/go/doc分割成[p go doc]；

然后更新增加路由的方法，通过树结构来存放路由；

之后就是获取路由信息，即将输入路由与树中的结构进行匹配，并按照一些方法得到对应的节点信息和附加信息，例如/p/go/doc匹配到/p/:lang/doc，解析结果为：{lang: “go”}，/static/css/geektutu.css匹配到/static/*filepath，解析结果为{filepath: “css/geektutu.css”}。

最后更新路由处理方式，将附加信息添加至Context中:Params map[string]string，并通过方法来调用该附加信息：

func (c *Context) Param(key string) string {
	value, _ := c.Params[key]
	return value
}
//解析路由
func (r *router) parsePattern(pattern string) []string {
	// 将pattern按照/来分割得到字符串集合
	vs := strings.Split(pattern, "/")

	//存放经过筛选的vs
	parts := make([]string, 0)

	for _, item := range vs {
		if item != "" {
			parts = append(parts, item)
			//仅第一个*开头的字符串会被存入数组中
			if item[0] == '*' {
				break
			}
		}
	}
	return parts
}
//添加路由并将路由插入到树中
/*
method:请求方法
patten:树中定义的路径
handler:处理方法
*/
func (r *router) addRoute(method string, pattern string, handler HandlerFunc) {
	parts := r.parsePattern(pattern)

	key := method + "-" + pattern
	//判断是否存在method对应的根节点，没有则初始化
	if _, ok := r.roots[method]; !ok {
		r.roots[method] = &node{}
	}
	//插入子节点到根节点下
	r.roots[method].insert(pattern, parts, 0)
	//根据路径存放对应的处理方法
	r.handlers[key] = handler
}
//查询路由
/*
method:请求方法
path:请求路径
*/
func (r *router) getRoute(method string, path string) (*node, map[string]string) {
	//路由集合
	searchParts := r.parsePattern(path)
	//用于存储*和:的信息
	params := make(map[string]string)
	//根节点
	root, ok := r.roots[method]
	//如果没有创建该根节点则返回nil
	if !ok {
		return nil, nil
	}

	//找出路由对应的节点
	n := root.search(searchParts, 0)

	if n != nil {
		//将树的路由切割成集合
		parts := r.parsePattern(n.pattern)
		//遍历路径，若存在:和*则进行处理
		for index, part := range parts {
			//存放:节点对应的路由名,例如/p/go/doc匹配到/p/:lang/doc，解析结果为：{lang: "go"}
			if part[0] == ':' {
				params[part[1:]] = searchParts[index]
			}
			//存放*节点之后的路由，例如/static/css/geektutu.css匹配到/static/*filepath，解析结果为{filepath: "css/geektutu.css"}
			if part[0] == '*' && len(part) > 1 {
				params[part[1:]] = strings.Join(searchParts[index:], "/")
				break
			}
		}
		return n, params
	}
	return nil, nil
}
/*
执行处理逻辑
*/
func (r *router) handle(c *Context) {
	n, params := r.getRoute(c.Method, c.Path)
	if n != nil {
		c.Params = params
		key := c.Method + "-" + n.pattern
		r.handlers[key](c)
	} else {
		c.String(http.StatusNotFound, "404 NOT FOUND: %s\n", c.Path)
	}
}

第四天 分组控制

有些时候不同的路由需要不同的处理方法，且路由中的子路由的处理方法也可能不同，这时就需要分组，例如/post是一个分组，/post/a和/post/b可以是该分组下的子分组，这时就可以用三种处理方法分别处理这三种路由。

分组控制的实现方式就是：定义一个结构体来存储这个组，然后由这个组来进行拓展。这个结构体所需要的功能有：前缀，用于保存当前路由以及拼接后续路由，例如前缀为/post，这时再来一个路由/a，拼接后的前缀就为/post/a；这个结构体需要调用路由的处理方法，因此需要有其他结构体包含在内；有时不同的路由需要单独实现处理方法，这时就需要中间件，例如/admin的分组，可以应用鉴权中间件；/分组应用日志中间件，/是默认的最顶层的分组，也就意味着给所有的路由，即整个框架增加了记录日志的能力。

/*
前缀prefix
中间件middlewares
路由调度Engine
*/
type RouterGroup struct {
	prefix      string
	middlewares []HandlerFunc
	engine      *Engine
}

为了让Engine结构体能够成为顶层的结构，因此结构体内需要实现该结构体：

Engine struct {
	*RouterGroup
	router *router
	groups []*RouterGroup // store all groups
}

groups表示囊括所有分组。

对Engine初始化，得到：

//初始化
func New() *Engine {
	engine := &Engine{router: newRouter()}
	engine.RouterGroup = &RouterGroup{engine: engine}
	engine.groups = []*RouterGroup{engine.RouterGroup}
	return engine
}

现在原来的一些方法可以用RouterGroup来调用：

//从当前分组中创建子分组，例如/post分组添加子分组/post/a
//所有分组共享一个Engine
func (group *RouterGroup) Group(prefix string) *RouterGroup {
	engine := group.engine
	newGroup := &RouterGroup{
		//拼接地址，例如父分组/post与子分组/a拼接得到新分组/post/a
		prefix: group.prefix + prefix,
		engine: engine,
	}
	engine.groups = append(engine.groups, newGroup)
	return newGroup
}

//添加路由
func (group *RouterGroup) addRoute(method string, comp string, handler HandlerFunc) {
	//主分组加目标分组
	pattern := group.prefix + comp
	log.Printf("Route %4s - %s", method, pattern)
	group.engine.router.addRoute(method, pattern, handler)
}

//请求方法
func (group *RouterGroup) GET(pattern string, handler HandlerFunc) {
	group.addRoute("GET", pattern, handler)
}

func (group *RouterGroup) POST(pattern string, handler HandlerFunc) {
	group.addRoute("POST", pattern, handler)
}

其中，Group方法用于创建当前分组的子分组并返回结构指针。

第五天 中间件

中间件与路由的处理方法不一样，中间件的作用在于能够为特定的路由添加功能，在上一节分组控制中，我们为不同的路由进行分组，现在为不同的分组来定义中间件。

Gee 的中间件的定义与路由映射的 Handler 一致，处理的输入是Context对象。中间件的插入点在初始化Context对象后，允许用户使用自己定义的中间件做一些额外的处理，例如记录日志等，以及对Context进行二次加工。并且可以使用(*Context).Next()函数来等用户自己定义的 Handler处理结束后，再做一些额外的操作，例如计算本次处理所用时间等。

即 Gee 的中间件支持用户在请求被处理的前后，做一些额外的操作。

其中Next函数的定义为：

func (c *Context) Next() {
	c.index++
	//中间件的数量
	s := len(c.handlers)
	//依次执行中间件
	for ; c.index < s; c.index++ {
		c.handlers[c.index](c)
	}
}

c.index表示中间件的索引，默认值为-1，因此调用Next函数会先增加index的值，然后依次调用中间件。

Context结构体增加中间件数组和索引以存放中间件：

//middleWare
	handlers []HandlerFunc //存放中间件和路由的处理方法
	index    int

然后路由分组RouterGroup增加Use函数来添加中间件：

//添加中间件到分组中
func (group *RouterGroup) Use(middlewares ...HandlerFunc) {
	group.middlewares = append(group.middlewares, middlewares...)
}

随后在初始化Context对象时根据条件将分组中存放的中间件赋值给对象：

func (engine *Engine) ServeHTTP(w http.ResponseWriter, req *http.Request) {
	var middlewares []HandlerFunc
	for _, group := range engine.groups {
		//中间件的使用条件，此处通过判断分组的前缀与输入路由的地址是否相符
		if strings.HasPrefix(req.URL.Path, group.prefix) {
			middlewares = append(middlewares, group.middlewares...)
		}
	}
	c := newContext(w, req)
	//存储中间件
	c.handlers = middlewares
	engine.router.handle(c)
}

此处根据分组的前缀与输入路由的前缀来将该分组的中间件进行传递。

最后为了统一管理，handle 函数中，将从路由匹配得到的 Handler 添加到 c.handlers列表中，执行c.Next()。

func (r *router) handle(c *Context) {
	n, params := r.getRoute(c.Method, c.Path)
	if n != nil {
		c.Params = params
		key := c.Method + "-" + n.pattern
		//将路由的处理方法加入到中间件数组中
		c.handlers = append(c.handlers, r.handlers[key])
	} else {
		//将不存在该路由的处理方法加入到数组中并调用
		c.handlers = append(c.handlers, func(c *Context) {
			c.String(http.StatusNotFound, "404 NOT FOUND: %s\n", c.Path)
		})
	}
	//调用中间件函数
	c.Next()
}

在ServeHTTP()中，每接收到一个请求都需要逐一匹配前缀来添加中间件，对程序性能有着负影响。
gin是在前缀树的节点中添加中间件的切片，这样在匹配动态路由并解析参数时，就可以同时获得各分组的中间件。因此基于此思想，对中间件的存取进行改进：

//添加路由
func (group *RouterGroup) addRoute(method string, comp string, handler HandlerFunc) {
	handlers := make([]HandlerFunc, len(group.middlewares))
	copy(handlers, group.middlewares)
	handlers = append(handlers, handler) // 将处理请求的函数与中间件函数结合
	//主分组加目标分组
	pattern := group.prefix + comp
	log.Printf("Route %4s - %s", method, pattern)
	group.engine.router.addRoute(method, pattern, handlers)
}

在结构中添加路由时将中间件一起添加进去，这样就避免进行遍历且能同时将中间件和请求函数合并在一起管理。

第六天 模板

服务器渲染页面，需要读取服务器上的文件，而客户端请求的文件地址与服务器上的文件地址不同，因此需要进行转换。例如客户端请求的文件地址为：/assets/filepath，而服务器的文件地址为/static/filepath，因此需要用static来替换assets。于是对服务器端增加两个方法：

//创建动态路由请求函数
func (group *RouterGroup) createStaticHandler(relativePath string, fs http.FileSystem) HandlerFunc {
	absolutePath := path.Join(group.prefix, relativePath) //将静态文件的相对地址与前缀拼接
	/*
		http.StripPrefix返回http.FileServer的结果：一个处理器Handler，将URL中的前缀中的prefix字符串删除，然后再交给后面的Handler处理
		http.FileServer表示将fs把之前定义的前缀进行替换
		例如：http.StripPrefix("/tmpfiles/", http.FileServer(http.Dir("/tmp")))，"/tmp"将"/tmpfiles/"进行替换
	*/
	fileServer := http.StripPrefix(absolutePath, http.FileServer(fs))
	return func(c *Context) {
		file := c.Param("filepath")
		//打开目标文件，如果不存在则返回错误状态码
		if _, err := fs.Open(file); err != nil {
			c.Status(http.StatusNotFound)
		}
		//文件处理逻辑
		fileServer.ServeHTTP(c.Writer, c.Req)
	}
}

//服务器静态文件
/*
relativePath:相对地址，例如相对地址为"/assets"，而绝对地址为：/assets/js/geektutu.js
root：目标地址，即将相对地址进行替换的相对地址，例如"/usr/geektutu/blog/static"替换"/assets"
用户访问localhost:9999/assets/js/geektutu.js，最终返回/usr/geektutu/blog/static/js/geektutu.js。
*/
func (group *RouterGroup) Static(relativePath string, root string) {
	//将磁盘上的某个文件夹root映射到路由relativePath，换句话说就是请求的地址为/assets/js/geektutu.js，
	//现在替换成/usr/geektutu/blog/static/js/geektutu.js来读取geektutu.js文件
	handler := group.createStaticHandler(relativePath, http.Dir(root))
	urlPattern := path.Join(relativePath, "/*filepath")
	group.GET(urlPattern, handler)
}

第一个方法就是用服务器的路由前缀地址来替代客户端的路由前缀地址，并针对该路径创建路由处理方法：利用http.StripPrefix来消除路由前缀，并用fs来替代且返回一个handler。

第二个方法就是对文件请求设置请求方法。

对文件路径处理之后，就需要通过模板来进行调用渲染。html/template为 HTML 提供了较为完整的支持。包括普通变量渲染、列表渲染、对象渲染等。于是对Engine结构增加两个属性:

htmlTemplates *template.Template // for html render
funcMap       template.FuncMap   // for html render

其中，htmlTemplates用于定义、调用、渲染各种模板，funcMap用于存储自定义的渲染函数：

func (engine *Engine) SetFuncMap(funcMap template.FuncMap) {
	engine.funcMap = funcMap
}

然后创建模板对象并解析模板内容：

/*
New：创建模板对象，参数为模板名，1个模板对象可以有多个name，通过t和各自的name，可以调用到指定的模板。
Funcs:将自定义函数与模板关联，需要在解析内容之前进行定义
ParseGlob:解析目录下的一个或多个模板的内容，并将这些内容关联到t上
*/
func (engine *Engine) LoadHTMLGlob(pattern string) {
	engine.htmlTemplates = template.Must(template.New("").Funcs(engine.funcMap).ParseGlob(pattern))
}

LoadHTMLGlob方法表示用template.New（参数表示模板名字，可定义多个名字）来创建模板对象，再通过Funcs来将自定义函数注入到模板中，然后ParseGlob来解析目标路径的所有模板，最后使用template.Must对模板对象进行封装。

创建了模板对象后就需要执行模板，于是对(*Context).HTML()进行修改：

//输出页面信息
func (c *Context) HTML(code int, name string, data interface{}) {
	c.SetHeader("Content-Type", "text/html")
	c.Status(code)
	//选择给定名字的模板并进行渲染写入Writer中
	if err := c.engine.htmlTemplates.ExecuteTemplate(c.Writer, name, data); err != nil {
		c.Fail(500, err.Error())
	}
}

增加了htmlTemplates.ExecuteTemplate函数以执行模板，该函数通过指定模板名字将data数据传入到Writer响应界面。由于调用的是engine的方法，因此需要在Context结构里添加属性以调用方法：

//创建Engine对象

第七天 错误处理

有时程序会因为一些数组越界，空指针等异常使得程序报错。因此需要引入错误处理，即出现错误后实施措施，例如使用recover函数来恢复程序：

func Recovery() HandlerFunc {
	return func(c *Context) {
		defer func() {
			if err := recover(); err != nil {
				message := fmt.Sprintf("%s", err)
				log.Printf("%s\n\n", trace(message))
				c.Fail(http.StatusInternalServerError, "Internal Server Error")
			}
		}()

		c.Next()
	}
}

程序引入一个中间件来使用 defer 挂载上错误恢复的函数，该函数的作用为跟踪报错的信息直到出错的位置，然后自定义错误内容输出到客户端，trace函数用于跟踪错误信息并将信息打印至服务端：

func trace(message string) string {
	var pcs [32]uintptr
	/*
		Callers 用来返回调用栈的程序计数器, 第 0 个 Caller 是 Callers 本身，第 1 个是上一层 trace，第 2 个是再上一层的 defer func。
	*/
	n := runtime.Callers(3, pcs[:]) //跳过前3个Caller

	var str strings.Builder
	str.WriteString(message + "\nTraceback:")
	for _, pc := range pcs[:n] {
		fn := runtime.FuncForPC(pc)   //获取对应的函数
		file, line := fn.FileLine(pc) // 获取到调用该函数的文件名和行号，打印在日志中。
		str.WriteString(fmt.Sprintf("\n\t%s:%d", file, line))
	}
	return str.String()

}
